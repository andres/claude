# Amendment: Export All AsyncReports

## Amends: `unified_client_export.md`

---

## Overview

Add a standalone shell-executable class `Export::AsyncReports` that dumps every completed
`AsyncReport` ever generated by the system into a flat folder. This is independent of the
client export — it can be run on its own or as an additional step.

---

## Usage (Rails console / shell)

```ruby
load 'app/lib/export/async_reports.rb'

Export::AsyncReports.run(destination: '/export/async_reports')
```

---

## Output Structure

All files land in a single flat directory. No subfolders.

```
/export/async_reports/
  export_log.txt
  2024-01-15_client_roster_Client_Roster.csv
  2024-01-20_aging_report_Aging_Dynamic_Report.csv
  2024-02-05_charges_Comprehensive_Charge_Report.csv
  2024-02-10_appointments_Appointments.csv
  2024-03-01_db_audit_DB_Audit.csv
  2024-03-02_demographics_Demographics.csv
  2023-11-01_nm_measure_New_Mexico_Measures.zip
  ...
```

**File naming convention:**

```
YYYY-MM-DD_{report_type}_{Human Name}.{ext}
```

Built from:
- `async_report.finished_at.strftime('%Y-%m-%d')`
- `async_report.stored_report.report_type`
- `AsyncReport.name_map[report_type]` converted to snake_case (spaces to underscores, strip special chars)
- `async_report.stored_report.report_file_type` for the extension

Example: a `client_roster` report finished on 2024-01-15 → `2024-01-15_client_roster_Client_Roster.csv`

If two reports of the same type share the same date, append the `AsyncReport#id` to avoid
collisions: `2024-01-15_client_roster_Client_Roster_4821.csv`

---

## Source Model

`AsyncReport` (`app/models/async_report.rb`)

Key attributes:

| Attribute | Description |
|-----------|-------------|
| `report_type` | Delegated to `stored_report.report_type` — string key e.g. `"client_roster"` |
| `finished_at` | `nil` if still running; present when complete |
| `with_errors` | Boolean — true if the run failed |
| `document` | Paperclip attachment (S3) — primary file storage |
| `stored_report.report_data` | Fallback text storage when no S3 document exists |
| `stored_report.report_file_type` | `"csv"`, `"zip"`, or `"xml"` |

Relevant scopes already on the model:

```ruby
AsyncReport.completed   # finished_at IS NOT NULL, ordered by finished_at DESC
AsyncReport.no_error    # with_errors IS FALSE
```

Use both:

```ruby
AsyncReport.completed.no_error.find_each
```

## Retrieving File Content

`AsyncReport` already has a `document_content` method:

```ruby
def document_content
  if document.exists?
    File.read(Paperclip.io_adapters.for(document).path)
  else
    stored_report.report_data
  end
end
```

When `document.exists?` is true, prefer copying directly via Paperclip to avoid loading
the file into memory:

```ruby
async_report.document.copy_to_local_file(:original, destination_file_path)
```

Fall back to writing `stored_report.report_data` when `document.exists?` is false.

---

## What to Skip

| Condition | Action |
|-----------|--------|
| `finished_at IS NULL` | Skip — still running |
| `with_errors = true` | Skip — failed run |
| `document.exists? == false` AND `stored_report.report_data.blank?` | Skip — no content; log it |

## Developer Notes

- `AsyncReport` records can be numerous — always use `find_each`, never `all.each`.
- Some reports produce ZIP or XML (`report_file_type` of `"zip"` or `"xml"`). The naming
  and copy logic is identical — the extension comes from `stored_report.report_file_type`.
- `AsyncReport.name_map` on the model maps every `report_type` key to its human name.
  Use this for the filename. If a key is missing from the map, fall back to the raw
  `report_type` string.
- Do not re-run or regenerate any reports — only export what already exists.
- This class is fully independent of `Export::Clients::Bundle`. It does not need to be
  called from there unless explicitly wired in later.
